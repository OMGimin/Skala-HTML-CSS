# SKALA HTML/CSS/JavaScript 과제 결과물 평가

## 1. 과제 진행 방향

이번 과제는 HTML, CSS, JavaScript 수업에서 배운 내용을 단계별로 적용하면서 하나의 실습 사이트 형태로 정리하는 것을 목표로 했다.
처음에는 `index.html` 한 파일에서 기본 HTML 구조와 자기소개, 링크 목록을 작성했고, 이후 `signup.html`, `travel.html`, `products.html`, `todo.html`처럼 기능별 페이지를 추가하면서 점점 확장하는 방식으로 진행했다.

React, Vue, TypeScript, Tailwind 같은 프레임워크는 사용하지 않고, 수업 자료에서 다룬 기본 태그와 CSS 속성, JavaScript 문법을 우선 사용했다.
다만 종합실습 단계에서는 결과물이 너무 단순해 보이지 않도록 hover, fixed navigation, dark mode, localStorage, fetch 같은 내용을 추가로 적용했다.

GitHub 주소: https://github.com/OMGimin/Skala-HTML-CSS

## 2. 전체 결과물 평가

전체적으로 HTML, CSS, JavaScript의 역할을 나누어 이해하는 데 도움이 된 과제였다.
HTML에서는 페이지 구조와 시맨틱 태그를 잡고, CSS에서는 레이아웃과 반응형, hover 효과를 적용했으며, JavaScript에서는 입력값 처리, DOM 조작, 이벤트 처리, 저장 기능을 구현했다.

초반 실습은 강의자료 범위 안에서 간단하게 작성했고, 종합실습으로 넘어가면서 페이지 디자인과 기능을 조금씩 보강했다.
처음부터 완성형 사이트를 만든 느낌보다는, 실습 내용을 바탕으로 기능을 하나씩 붙여가며 발전시킨 결과물이라고 볼 수 있다.

아쉬운 점은 모든 페이지의 디자인을 완전히 동일한 수준으로 맞추지는 못했다는 점이다.
그래도 메인 페이지, 회원가입 페이지, 여행지 페이지, 상품 카드 갤러리, 할 일 관리 앱은 서로 이동할 수 있도록 연결했고, 전체 과제 결과물을 하나의 사이트처럼 확인할 수 있게 구성했다.

## 3. 실습별 구현 및 평가

### Day 1 기본 HTML 실습

`index.html`에서 HTML 기본 구조, 자기소개 문단, 목록, 외부 링크를 작성했다.
`h1`, `h2`, `p`, `ul`, `ol`, `li`, `a href`, `target="_blank"`를 사용했고, `list-style-type`도 적용했다.

처음에는 단순한 자기소개 페이지였지만, 이후 다른 실습 페이지로 이동할 수 있는 메인 허브 역할로 확장했다.
이 부분은 기본 태그를 연습하면서도 최종 제출물의 입구 역할을 하도록 구성했다는 점이 좋았다.

### 회원가입 Form 및 결과 페이지

`signup.html`과 `result.html`을 만들어 회원가입 입력값을 결과 페이지에서 table로 확인할 수 있게 했다.
`form`, `label`, 다양한 `input type`, `radio`, `checkbox`, `select`, `textarea`, `submit`, `reset`을 사용했다.

유효성 검증으로는 `required`, `pattern`, `minlength`, `maxlength`, `min`, `max`를 적용했다.
아이디는 영문/숫자 4~12자, 비밀번호는 8자 이상, 전화번호는 `010-0000-0000` 형식으로 제한했다.
결과 페이지에서는 `URLSearchParams`를 이용해 입력값을 읽고, 관심분야처럼 여러 개 선택되는 값은 함께 표시되도록 처리했다.

개인정보 보호를 위해 비밀번호 값 자체는 결과 페이지에 직접 보여주지 않고, 입력 여부만 표시하도록 했다.
이 부분은 요구사항에는 없었지만 실제 페이지라면 비밀번호를 그대로 보여주는 것이 어색하다고 판단해서 추가로 반영했다.

### 여행지 소개 페이지

`travel.html`은 `header`, `nav`, `main`, `section`, `article`, `aside`, `footer` 구조로 작성했다.
상단 내비게이션은 `position: fixed`와 `z-index`로 고정했고, 앵커 링크를 이용해 페이지 안의 섹션으로 이동할 수 있게 했다.

여행지는 부산, 제주, 오사카를 중심으로 구성했고, `figure`와 `figcaption`을 사용해 이미지와 설명을 묶었다.
모든 이미지에는 `alt`를 작성했고, 카드에 마우스를 올리면 이미지가 확대되도록 hover 효과를 적용했다.
히어로 영역에는 배경 이미지와 gradient overlay를 사용해 첫 화면의 완성도를 높였다.

또한 `article`과 `aside`를 활용해 먹거리 소개와 여행 팁을 2단 구조로 배치했다.
영상은 `video`, `controls`, `poster`, `source`를 사용해 수업 요구사항에 맞게 구성했다.
처음에는 YouTube 영상 링크를 사용하는 방법도 고려했고 실제로 적용해보기도 했지만, 과제 요구사항이 `video` 태그와 `source` 사용에 더 가깝다고 판단했다.
그래서 최종적으로는 외부 YouTube 링크 대신 로컬 영상 파일을 `video` 태그로 연결하는 방식으로 다시 수정했다.
다만 영상 파일은 용량이 커서 GitHub에는 제외했고, Google Drive 제출 시에는 `media` 폴더의 영상 파일을 함께 제출해야 정상 재생된다.

### JavaScript 기초 및 DOM/Event 실습

JavaScript 기초 실습에서는 입력값을 받아 조건문과 배열을 사용해 결과를 화면에 표시했다.
단순히 콘솔에 출력하는 것에서 끝내지 않고, 사용자가 입력한 후 결과를 직접 확인할 수 있도록 구성했다.

DOM/Event 실습에서는 버튼 클릭 시 텍스트 변경, 입력값 읽기, 목록 추가와 삭제를 구현했다.
`addEventListener`, `textContent`, `classList`, `createElement` 등을 사용해 HTML 요소를 JavaScript로 다루는 흐름을 연습했다.

Async/Module 실습에서는 `import/export`를 사용해 데이터를 분리하고, 비동기처럼 시간이 지난 뒤 데이터가 표시되는 흐름을 확인했다.
Wrap-up Quiz는 클릭형 문제 풀이 형태로 만들어 정답/오답 확인이 가능하도록 했다.

### 반응형 상품 카드 갤러리

`products.html`에서는 여행 상품 카드 갤러리를 만들었다.
기본 요구사항보다 확장해서 상품 카드를 9개로 늘렸고, 부산, 제주, 오사카뿐 아니라 서울, 도쿄, 방콕 등 다양한 여행지를 포함했다.

CSS에서는 `:root`에 색상 변수를 정의하고 `var()`로 사용했다.
상품 카드 배치는 CSS Grid로 3열을 기본으로 구성했고, `@media`를 사용해 900px 이하에서는 2열, 600px 이하에서는 1열로 바뀌도록 했다.
상단바는 Flexbox로 로고는 왼쪽, 메뉴와 다크모드 버튼은 오른쪽에 오도록 정렬했다.

카드에 hover하면 위로 떠오르고 그림자가 진해지며 이미지가 확대된다.
히어로 제목과 부제는 아래에서 순차적으로 등장하도록 `@keyframes` 애니메이션을 적용했다.
카드들도 새로고침 시 순서대로 나타나도록 animation delay를 주었다.

추가로 다크모드 버튼을 만들어 배경, 카드, 글자색이 바뀌도록 했다.
또한 OS에서 "동작 줄이기" 설정을 켠 경우 애니메이션이 꺼지도록 `prefers-reduced-motion`을 적용했다.

### 인터랙티브 할 일 관리 앱

`todo.html`, `js/todo.js`, `js/storage.js`를 만들어 할 일 관리 앱을 구현했다.
할 일 하나는 `{ id, text, done }` 형태의 객체로 관리했고, 전체 목록은 배열로 저장했다.

기능은 할 일 추가, Enter 키 추가, 빈 값 방지, 완료 체크, 삭제, 전체/진행중/완료 필터, 전체/진행중/완료 개수 표시로 구성했다.
동적으로 생성되는 목록은 부모 `ul`에 이벤트 리스너를 한 번만 등록하고 `closest()`로 대상을 찾는 이벤트 위임 방식을 사용했다.

`localStorage`를 이용해 새로고침 후에도 할 일이 유지되도록 했다.
`storage.js`에는 저장과 불러오기 함수를 분리하고, `todo.js`에서는 화면 렌더링과 이벤트 처리를 담당하도록 나누었다.
`type="module"`을 사용했기 때문에 Live Server 또는 `python -m http.server` 같은 로컬 서버에서 실행해야 한다.

오늘의 한마디는 `fetch`, `async/await`, `try/catch`를 사용해 `data/quotes.json`에서 불러오도록 했다.
파일을 찾지 못하거나 요청이 실패하면 기본 문구가 표시되도록 처리했다.

## 4. 조건에 없었지만 추가로 구현한 내용

### 메인 허브 페이지 구성

각 실습 페이지가 따로 흩어져 보이지 않도록 `index.html`을 메인 허브 페이지로 확장했다.
자기소개, 실습 페이지 바로가기, 앞으로의 계획, GitHub 링크를 한 화면에서 확인할 수 있게 했다.
제출물을 확인하는 사람이 여러 HTML 파일을 직접 찾아 열지 않아도 되도록 한 점을 추가로 신경 썼다.

### 페이지 간 이동 버튼

`signup.html`, `result.html`, `travel.html`, `products.html`, `todo.html` 등 주요 페이지에 메인으로 돌아가는 버튼을 추가했다.
처음에는 링크만 있었지만, 사용자가 보기에는 버튼 형태가 더 명확하다고 생각해 `link-button` 스타일을 만들어 적용했다.

### 전체 색상 통일

회원가입 페이지에서 사용한 파란색 계열을 메인 색상으로 정하고, 다른 페이지에도 비슷한 색을 적용했다.
완전히 전문적인 디자인 시스템처럼 만들지는 않았지만, 페이지마다 너무 따로 노는 느낌을 줄이기 위해 제목, 버튼, 강조 색을 맞췄다.

### 디자인 품질 개선

조원들의 결과물을 참고해 단순한 HTML 문서처럼 보이는 부분을 개선했다.
히어로 영역, 카드 레이아웃, fixed navigation, footer, hover 효과를 추가해 과제 결과물이 조금 더 완성된 웹페이지처럼 보이도록 했다.
다만 수업 범위를 벗어나지 않도록 복잡한 라이브러리나 프레임워크는 사용하지 않았다.

### 상품 카드 갤러리 확장

필수 요구사항은 상품 카드 6개 이상이었지만, 여행지 이미지를 추가로 준비해 총 9개의 카드로 확장했다.
다양한 여행지를 보여주면 카드 갤러리의 반응형 배치와 hover 효과를 더 확실하게 확인할 수 있다고 판단했다.

### 접근성 관련 보완

이미지에는 `alt`를 작성했고, 입력 요소에는 `label`을 연결했다.
상품 카드 갤러리에서는 `prefers-reduced-motion`을 적용해 사용자가 OS에서 애니메이션 줄이기를 설정한 경우 움직임이 줄어들도록 했다.
필수 조건은 아니었지만, 사용자 환경을 고려한 추가 구현으로 볼 수 있다.

### GitHub 용량 제한 대응

여행지 페이지에 로컬 영상을 사용했지만, mp4 파일이 GitHub의 100MB 제한을 초과했다.
그래서 `.gitignore`에 `media/*.mp4`를 추가해 GitHub에는 대용량 영상이 올라가지 않도록 처리했다.
이 과정에서 GitHub에는 소스코드와 이미지 중심으로 업로드하고, 영상 파일은 별도 제출물로 관리해야 한다는 점을 알게 되었다.

### 영상 적용 방식 변경

여행지 페이지의 영상은 처음에 YouTube 링크로 연결하는 방식을 시도했다.
하지만 종합실습 요구사항에 `video controls`, `poster`, `source`가 포함되어 있었기 때문에, 단순 외부 링크나 iframe보다 HTML5 `video` 태그를 직접 사용하는 방식이 과제 방향에 더 맞다고 판단했다.
그래서 YouTube 방식은 제거하고, 직접 준비한 영상을 `media/Osaka.mp4`로 연결해 `controls`, `poster`, `source`를 확인할 수 있도록 수정했다.

## 5. 실행 및 검증 내용

실행 결과 캡처는 실습 흐름에 맞춰 `screenshots` 폴더에 정리했다.
회원가입 페이지는 필수값 검증, 패턴 검증, 정상 제출 결과를 확인했고, 여행지 페이지는 fixed navigation, 이미지 hover, 영상 영역, footer를 확인했다.

상품 카드 갤러리는 라이트모드, 다크모드, 반응형 3열/2열/1열, 카드 hover, 등장 애니메이션을 확인했다.
할 일 관리 앱은 추가, 완료, 삭제, 필터, 새로고침 후 localStorage 유지, 비동기 문구 로드와 실패 시 기본 문구 표시를 확인했다.

## 6. 아쉬운 점과 보완하고 싶은 점

이번 과제에서는 수업에서 배운 내용을 실제 페이지에 적용해보는 데 집중했기 때문에, 코드 구조가 완전히 깔끔하게 정리되지는 않았다.
특히 CSS가 하나의 `style.css`에 많이 모여 있어 페이지가 더 많아지면 유지보수가 어려워질 수 있다고 느꼈다.

다음에 보완한다면 페이지별 CSS 파일을 분리하거나, 공통 스타일과 개별 페이지 스타일을 나눠 관리해보고 싶다.
또한 상품 카드 데이터나 여행지 데이터도 HTML에 직접 반복해서 쓰기보다 JavaScript 배열 또는 JSON으로 분리하면 수정이 쉬울 것 같다.

JavaScript에서는 기본 기능 구현에 집중했지만, 오류 상황에 대한 안내 메시지나 사용자 경험을 더 다듬을 수 있을 것 같다.
예를 들어 할 일 관리 앱에서 전체 삭제, 완료 항목만 삭제, 수정 기능을 추가하면 실제로 더 사용하기 좋은 앱이 될 것 같다.

## 7. 최종 소감

이번 과제를 진행하면서 HTML은 구조, CSS는 화면 표현, JavaScript는 동작을 담당한다는 점을 직접 확인할 수 있었다.
처음에는 단순한 태그 작성부터 시작했지만, 최종적으로는 회원가입 결과 처리, 여행지 소개, 반응형 카드 갤러리, 할 일 관리 앱까지 구현하면서 각 기술이 어떻게 연결되는지 이해할 수 있었다.

아직 코드가 완벽하다고 보기는 어렵지만, 수업에서 배운 내용을 실제 결과물로 만들어보며 부족한 부분을 확인했다는 점에서 의미가 있었다.
